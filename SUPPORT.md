# Getting support

## Resources

| Resource | URL |
|----------|-----|
| Documentation | Each repository's `docs/` folder or GitHub Pages site |
| Discussions | [github.com/orgs/Presidencia25/discussions](https://github.com/orgs/Presidencia25/discussions) |
| Email (general) | **tecnologia@presidencia.gob.mx** |
| Security issues | See [SECURITY.md](SECURITY.md) — **do not** open a public issue |

## Before asking for help

1. Search the repository's existing [issues](../../issues) and [discussions](../../discussions).
2. Read the relevant documentation in the `docs/` folder.
3. Check whether the issue is already fixed in the latest release.

## Opening a support issue

If you cannot find an answer, open an issue in the relevant repository and use the **Bug Report** or **Question** template. Please provide:

- A clear description of the problem
- Steps to reproduce (for bugs)
- Expected vs actual behaviour
- Relevant logs, screenshots, or error messages
- Version or commit SHA of the code you are using

## Response times

| Priority | Target first response |
|----------|-----------------------|
| Critical (service outage) | 4 business hours |
| High (major feature broken) | 1 business day |
| Normal | 3 business days |
| Low / questions | 5 business days |
